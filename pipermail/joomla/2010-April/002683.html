1.6 will not have many of the features of K2.  1.6 will have ACL's....

K2 has a user profile system, basic ACL, and a CCK.

That said, I have found it to be somewhat inflexible and javascript heavy
and while it is a great system, just like sobi and all the other CCK's I
find myself having to make too many hacks to be usable.

Also I really don't like an all in one integrated system....I prefer my user
profile system seperate from my other systems.

At this point, or at least right before the move and the fire, I was looking
into Phing to make a generic component creator[there are a couple online
ones but their not advanced enough for me...I want one that creates a basic
CRUD system for a basic 2-3 table layout]....

That way when I need to "clone" the component, I can just use the template
and create another one.

Use very basic database fields[text and integers is enough for most
applications, and custom coding can handle the complex cases].

Of course that project was shelved....along with my entire redmine project
system that I had setup on my laptop and had not had "time" to push to my
server yet.
