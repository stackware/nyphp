David,

I believe with a locally hosted video you place a .jpg with the same 
file base name as the video, and it becomes the poster image (image that 
displays before the video plays).

My first instinct would be to try placing a jpg with the same name as 
the YouTube video in the base folder for the locally hosted videos - as 
specified in the AllVideos plugin.

When I used AllVideosReloaded with Joomla 1.5, (either from their doc or 
the JW Player site - I don't remember which right off) code like this 
would work {m4v img="poster/testimonial-poster.jpg" width="440" 
height="266"}sbhc1{/m4v}

if the structure was /images/stories/video
then poster would be /images/stories/poster

Whether you can designate a poster image regardless of whether you are 
streaming a local or remote video I can't say. I never tried remote.

Hope this spares you the several bruises I suffered banging my head 
against the wall on this problem.
Mark
