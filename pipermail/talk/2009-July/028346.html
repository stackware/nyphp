I found my bug (actually my partner found it)- embarrassed to admit it but
here it is - the sql to parse is being stored in a var called $insSQL and
the oci_parse is using $insSql (note the QL vs ql) I knew it was something
dumb :-(. Interesting that the oci_parse() does not throw an error on a null
or empty var.

$insSQL = "INSERT INTO KB_TEST  VALUES (:TESTNAME)"; 
$stmt = oci_parse($con, $insSql); 
if (!$stmt) die ('Parse for insert failed'); 
$val = 'ERIC'; 
$bindVar = ':TESTNAME'; 
ioci_bind_by_name ($stmt, $bindVar, $val))
