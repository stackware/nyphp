Thanks to the people who kindly wrote to me about the binary data
issue.  We handled it by going to a work-around using
base64_encode/base64_decode.  This puts the binary data into text form
but requires another 33% storage space.  These functions were also
useful in another part of the system involving binary data, so we got
two with one.
