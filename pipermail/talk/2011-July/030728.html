is to set the expiration date to Jan 1, 2038.

>  PHP has some useful predefined constants, particularly PHP_INT_MAX in
> this case (http://php.net/manual/en/reserved.constants.php).  What you
> should really be doing while setting the cookie is ignoring time()
> completely, and just set the expiration time to the maximum size of an
> integer.
>
>   setcookie(“name”, “val”, PHP_INT_MAX);
>
> This way, the cookie expires in 2038 for everyone, which is the furthest
> expiration you can set anyway.
>
> On a more theoretical note, there really isn't a need to set a cookie's
> expiration date beyond, let's say 5 years.  For that cookie to even still be
> alive, you have to hope that the user:
> - is using the same computer for five years
> - has never reformatted the hard drive or re-installed a fresh OS
> - has never switched browsers
> - has never cleared their cookies
>
> I'd say a combination of all of the above is extremely rare, and I can't
> really think of a good use-case where a cookie would still be useful 5 years
> from now, given that the internet is constantly evolving.
