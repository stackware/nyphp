it works in conjunction with your host file

windows/system32/drivers/etc/hosts

in which case ur ip will be given a local name to use as part of the
"host:"
header to allow for multiple setups.  you need to edit the file in order
to
get a developmental virtual hosting setup running.

192.168.0.7    linuxbox

typing in "linuxbox" would resolve to 192.168.0.7 and

<VirtualHost *>
DocumentRoot "c:\somewhere"
ServerName linuxbox
</VirtualHost>

would let you map it back ... you can also use a mounted DocumentRoot
(x:,
y:, z:, etc)
