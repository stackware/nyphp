It's rare for a browser to send broken referers, but yeah,
there's
nothing to stop another server from calling spy.php with a
spoofed
referer and then doing whatever with the result. You might try
to
notice if the same client IP makes a number of requests using
different sitekeys...

There probably isn't much you can do that would be worth the
time it
took to implement. After all, even with perfect security on the
script, the numbers are still out there in public, ready to be
copy-and-pasted into a spreadsheet every morning.
