This is funny because I think one of the biggest warts with Java is
bloated poorly designed libs like NIO. As the author of JCIFS (the
Java CIFS networking client), which is pretty sophisticated about
multiplexing requests, I have some experience in this area and I have
found that Java is rather bad for writing efficient networking
clients. The limitations in the synchronized primitive is a problem
for us. We really need a simple condition variable but the concurrency
API bloated to the point where I want to implement a simple condvar
using synchronized but that's a non-trivial error prone exercise. It's
all very disappointing really since the language by itself is great.
