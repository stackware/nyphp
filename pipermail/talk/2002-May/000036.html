This post is basically meaningless, but I wanted to show off my amazing
Visio skills :->.  I've attached a .jpg that shows the basic structure
of NYPHP.org's backend, including list and authentication.

Right now, the site is "hosted" on two servers.  The .html is served
from nyphp.org, but the core code resides on a remote server.  There's
not much sense in this at this point, however, I'm curious as to
anyone's expierence with XMLRPC in PHP - solid, reliable implementation
yet?

providing an authentication/app framework to what could be any number
of Parasites.  I'm using a homegrown RPC structure now, however would
like to move to XMLRPC for standardization reasons.
