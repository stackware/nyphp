As always, the right tool for the right job. PHP and Java are >not<
competitors. I'm finding that PHP is, in addition to being a decent
open application platform, an appropriate tool for rapid prototyping
of Java applications. As long as you use classes, it's not terribly
difficult to port PHP apps to Java.

This would be made much much easier with two features which PHP lacks
(which have just become my top two feature requests):

1) Server-side request forwarding.

2) Private and protected instance variables.
