yes, I use mysql as well. it is better than oracle to handle mutiple languages from some point. Mysql is a databse that whatever you put in, you can get exactly the same out. you don't need to warry about charsets. If you will do some thing on Chinese and Japanese, it will have some troubles: Chinese has two charsets: GB2312 and Big5. Japanese also has more charsets. maybe you need two open source programs: recode and jcode to do the convert.
 Joseph Annino wrote:The MySQL docs have a whole chapter about these issues:
http://www.mysql.com/doc/en/Character_sets.html

I think MySQL handles these things in a simpler manner than Oracle.
