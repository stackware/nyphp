On Wed, Sep 18, 2013 at 11:14 AM, Joseph Crawford <codebowl at gmail.com>wrote:

> I am curious why people are suggestion to use a web service that adds to
> the monthly bill for the company rather than a simple print using
> javascript
>

I think I was the only one that suggested it. And I suggested it for this
flow:

1. Person places order online.
2. Server receives order, sends fax to pizza shop.
3. Pizza shop receives order as fax. No on-site computers are needed and no
need for them to check and print orders.

Interfax says "Free unlimited faxing to one designated fax number." It says
it has restrictions, but it might be possible to use them basically for
free depending on what those restrictions are.

But, if they can just log into an onsite computer and check orders and
print them, then yeah, that's way simpler and the same thing I did for a
client. It might be possible to create some type of application that
automates printing new orders, but that is probably going to take a decent
amount of development.
