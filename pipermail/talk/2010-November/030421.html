someday(it's LAMP related)

If you want apache to be able to read and write to a nfs share you need to
mount it with a context
mount -t nfs -o context=user_u:object_r:public_content_rw_t
192.168.199.32:/home/nfsShare /ordertmp/

Here is the solution.  Selinux will not allow httpd to connect to other
network servers by default.
The command to change this behavior is

setsebool -P httpd_can_network_connect=1

Everything works immediately and survives a reboot
